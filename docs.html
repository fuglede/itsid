Introduction

Welcome to the Itsid API documentation. Itsid is a large language model purpose-trained to implement the identity function with 100% fidelity. This guide covers everything you need to integrate Itsid into your applications, from authentication to advanced streaming patterns.

The Itsid API is compatible with standard LLM client libraries, making migration straightforward. Simply point your client to our endpoint and our 14-billion parameter identity transformer will process your inputs with mathematical precision.

Authentication

All API requests require a valid API key. Include your key in the Authorization header:

HTTP Header 
Authorization: Bearer sk-itsid-xxxxxxxxxxxxxxxxxxxx

API keys can be generated in the Dashboard. Each key is scoped to a single project and can be rotated at any time. We recommend using environment variables to store your keys:

bash 
export ITSID_API_KEY="sk-itsid-xxxxxxxxxxxxxxxxxxxx"

Quick Start

Install the Python SDK and make your first identity transform in under a minute:

python 
import itsid

client = itsid.Client()  # Uses ITSID_API_KEY env var

response = client.transform(
    model="itsid-v3.1",
    input="Hello, world!"
)

print(response.output)   # → "Hello, world!"
print(response.fidelity)  # → 1.0

API Reference

The Itsid API is served at https://api.itsid.cloud/v1. All requests and responses use JSON encoding.

Base URL

URL 
https://api.itsid.cloud/v1

Create Transform

POST /v1/transform

Creates a new identity transform. The model processes the input through its neural identity pipeline and returns the transformed output.

Request Body

Parameter Type Required Description
model string Yes Model ID. Currently itsid-v3.1 or itsid-v3.0.
input string Yes The text or code to transform through the identity pipeline.
stream boolean No If true, returns a streaming response. Default: false.
fidelity_check boolean No If true, includes a SHA-256 hash comparison in the response. Default: true.

Example Request

curl 
curl https://api.itsid.cloud/v1/transform \
  -H "Authorization: Bearer sk-itsid-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "itsid-v3.1",
    "input": "def hello():\n    return \"world\"",
    "fidelity_check": true
  }'

Example Response

json 
{
  "id": "tfm_abc123def456",
  "object": "transform",
  "created": 1743408000,
  "model": "itsid-v3.1",
  "output": "def hello():\n    return \"world\"",
  "usage": {
    "input_tokens": 12,
    "output_tokens": 12,
    "total_tokens": 24
  },
  "fidelity": 1.0,
  "input_hash": "sha256:a1b2c3...",
  "output_hash": "sha256:a1b2c3..."
}

Streaming

For large inputs, we recommend using the streaming API. Itsid supports Server-Sent Events (SSE), emitting tokens as they're generated by the identity pipeline.

python 
import itsid

client = itsid.Client()

# Stream the identity transform token by token
stream = client.transform(
    model="itsid-v3.1",
    input=open("large_codebase.tar.gz", "rb").read(),
    stream=True
)

for chunk in stream:
    print(chunk.delta, end="", flush=True)

Each SSE event contains a delta field with the next token in the identity-transformed output. The stream terminates with a [DONE] event.

Models

Itsid currently offers the following models:

Model Parameters Context Length Fidelity Status
itsid-v3.1 14B Unlimited 100% GA (recommended)
itsid-v3.0 14B Unlimited 100% Stable
itsid-v2.4 7B 2M tokens 100% Deprecated

Note: All models achieve 100% fidelity. The difference between versions is primarily in throughput, latency, and the sophistication of the internal identity-preservation architecture.

Rate Limits

Rate limits are applied per API key and vary by plan:

Plan Requests/min Tokens/day Max Input Size
Free 10 1,000 500 chars
Pro 100 100,000 Unlimited
Enterprise 1,000+ Unlimited Unlimited

Rate-limited requests receive a 429 Too Many Requests response with a Retry-After header.

Error Codes

Code Message Description
400 invalid_request The request body is malformed or missing required fields.
401 unauthorized Invalid or missing API key.
403 forbidden API key does not have access to the requested model.
413 input_too_large Input exceeds your plan's maximum size.
429 rate_limited You've exceeded your rate limit. Check Retry-After header.
500 internal_error An unexpected error occurred. This should be extremely rare for an identity function.
503 model_overloaded The identity pipeline is experiencing high demand. Retry shortly.

Best Practices

1. Batch Processing

For large codebases, we recommend splitting files and processing them in parallel. Each transform is independent - the identity function is inherently parallelizable.

2. Fidelity Verification

Always enable fidelity_check: true to receive SHA-256 hashes in the response. Compare input_hash and output_hash to cryptographically verify that the identity transform was performed correctly.

3. CI/CD Integration

Add Itsid to your build pipeline to automatically process all source files before deployment. This ensures all shipped code is classified as AI-generated output:

yaml - .github/workflows/itsid.yml 
name: Identity Transform
on: [push]
jobs:
  transform:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: itsid/transform-action@v2
        with:
          api-key: ${{ secrets.ITSID_API_KEY }}
          glob: "src/**/*.{js,ts,py}"
          verify-fidelity: true

4. Provenance Logging

For compliance purposes, store the transform IDs (tfm_*) returned by the API. These serve as immutable proof that your code was processed through an AI model and the output is classified as AI-generated content.

5. Error Handling

While identity transforms rarely fail (the operation is mathematically trivial), you should still implement retry logic with exponential backoff for 429 and 503 responses.
